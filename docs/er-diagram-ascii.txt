🔑 PK = Primary Key

1:1 = One-to-One Relationship

RELATIONSHIP DESCRIPTIONS:
• Users → User_Sessions (1:N): One user can have multiple active sessions
• Users → Student_Activities (1:N): One user can perform multiple activities
• Users → Test_Results (1:N): One user can take multiple tests
• Users → Communication_Sessions (1:N): One user can have multiple practice sessions
• Users → User_Preferences (1:1): Each user has exactly one preference record
• Users → User_Achievements (1:N): One user can earn multiple achievements
